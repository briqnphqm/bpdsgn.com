UI Chip & Number

Two small interactive controls used throughout the portfolio. The UI Chip acts as a filter that toggles visibility of a section; the Number is a numbered hotspot that, when clicked, swaps the content of an adjacent note card. Both are built on the same interaction model — an unselected default, a selected active, and a hover feedback state for each.

Component overview

Both components share a common state model (Unselected → Hover → Selected → Selected Hover) so users can transfer interaction knowledge between them. They differ in form factor and purpose.

UI Chip

A pill-shaped toggle used as a filter control. Clicking a chip shows or hides a section of a page. Chips are typed labels (e.g. Research, UI, Prototype) placed in a toolbar above the filtered content.

Number

A small circular hotspot anchored to a point on a visual (screenshot, diagram, map). Clicking a Number updates an accompanying note card to describe what that annotation is pointing to. Numbers are always grouped as a set (1 / 2 / 3…).

Why the Number grows on hover. Because the Number sits on top of a visual, it can be hard to spot. Enlarging on hover is the primary affordance that tells the user it is clickable — it stands in for an underline or color change on a text link.

Usage guidelines

UI Chip

✓ Do
  • Use a chip for filters that toggle visibility of a clearly labeled group of content.
  • Keep labels to a single word or two. Short labels read faster in a horizontal toolbar.
  • Show the chip group above the content it filters, not floating far from it.
  • Allow an All chip, or allow deselecting the active chip, so users can return to the unfiltered view.
✗ Don't
  • Don't use a chip for destructive or one-way actions (use a button instead).
  • Don't nest chips inside form controls — they are not a replacement for checkboxes.
  • Don't mix chip labels with counts unless every chip has one (keeps visual weight consistent).

Number

✓ Do
  • Place the Number directly on the region of the visual it references.
  • Number sequentially (1, 2, 3) following reading order of the visual.
  • Pair the Number group with a single note card; update the card's content on click.
  • Keep the first Number pre-selected on page load so the note card always shows content.
✗ Don't
  • Don't use more than ~5 Numbers on one visual; dense annotations become hard to scan.
  • Don't place Numbers over critical detail in the image — nudge them into the margin if needed.
  • Don't use Numbers as navigation between pages (use a stepper instead).

Accessibility guidelines

Both components must be operable with a keyboard and announce their state to assistive technology. The notes below apply to both unless called out.

Semantics & ARIA

Attribute Requirement
role Render as <button type="button">. Avoid <div> or <span> — native buttons get keyboard focus, Enter/Space activation, and correct roles for free.
aria-pressed Set true when selected and false when unselected. This tells screen readers the toggle state.
aria-label On Number, use aria-label="Annotation 1 of 3" (or similar) so the purpose is clear without the surrounding visual.
Group labeling Wrap a chip row in role="group" aria-label="Filters". Wrap a Number set in role="group" aria-label="Annotations".

Keyboard

Key Behavior
TabMoves focus to the control. A visible :focus-visible ring is required.
Shift + TabMoves focus back.
Enter / SpaceToggles selection and fires the update (filter for Chip, card content for Number).
← / →Optional on Number groups: moves selection to the previous/next Number, wrapping at the ends.

Color & contrast

  • Chip selected text #FFFFFF on #392A13 → ratio 13.8 : 1 (passes WCAG AAA).
  • Chip unselected text #5E523F on #FFFFFF → ratio 8.6 : 1 (passes WCAG AAA).
  • Selected Hover text #A49E95 on #392A13 is a styling hover — state is already conveyed by aria-pressed, so low contrast here is acceptable. Do not rely on it alone to signal selection.
  • Number default 10 px text is small; the 2 px minimum stroke of Manrope SemiBold on #90BDE4/#392A13 backgrounds meets 3 : 1 for non-text UI contrast.

Motion & reduced motion

The Number's hover scale uses a 140 ms ease transition. Respect prefers-reduced-motion: reduce by disabling the size transition so the component snaps between states instead of animating.

Touch targets

The default Number is only 26 px in diameter — smaller than the WAI-recommended 44 × 44 px target. On touch viewports, add a transparent 44 × 44 hit area around the Number (e.g. padding: 9px; background-clip: content-box) so it is tappable without enlarging the visual dot.

Quick audit checklist. Before shipping: native <button> · aria-pressed reflects state · visible focus ring · 44 px touch target on mobile · respects prefers-reduced-motion · state never conveyed by color alone.
